Customize of search algorithm

If you use accessors or mutators of SearchParams, you can change Sayuri's search algorithm.

By default, it's set up by my groundless intuition.

The following constant values are indexes of arrays.

  • NUM_PIECE_TYPES: A number of piece types.
    • EMPTY: It's not any piece.
    • PAWN: Pawn.
    • KNIGHT: Knight.
    • BISHOP: Bishop.
    • ROOK: Rook.
    • QUEEN: Queen.
    • KING: King.
  • NUM_SQUARES: A number of squares on the board.
    • A1: A1 square.
    • B1: B1 square.
    • C1: C1 square.
    • (Omitted from D1 to E8)
    • F8: F8 square.
    • G8: G8 square.
    • H8: H8 square.

Accessors and mutators

Material

  • material()
    • Set Materials of each piece type.
    • Not only used in search algorithm, but also used in evaluation function.

YBWC

  • ybwc_limit_depth()
    • The engine doesn't execute YBWC, if the depth less than this set value.
  • ybwc_after_moves()
    • The engine executes YBWC after this set value of candidate moves have been searched.

Aspiration Windows

  • enable_aspiration_windows()
    • Set whether to enable Aspiration Windows or not.
  • aspiration_windows_limit_depth()
    • The engine doesn't execute Aspiration Windows, if the depth less than this set value.
  • aspiration_windows_delta()
    • A basic value of how much size the engine expands search window.
    • If the engine have failed searching, it expands the window by "(This set value) * 2n".

SEE

  • enable_see()
    • Set whether to enable SEE or not.

History Heuristic

  • enable_history()
    • Set whether to enable History Heuristic or not.

Killer Move Heuristic

  • enable_killer()
    • Set whether to enable Killer Move Heuristic or not.
  • enable_killer_2()
    • Set whether to enable the killer move before 2 plies of the current searching node or not.

Transposition Table

  • enable_ttable()
    • Set whether to enable Transposition Table or not.

Internal Iterative Deepening

  • enable_iid()
    • Set whether to enable Internal Iterative Deepening or not.
  • iid_limit_depth()
    • The engine doesn't execute IID, if the depth less than this set value.
  • iid_search_depth()
    • IID searches in this set value as depth.

Null Move Reduction

  • enable_nmr()
    • Set whether to enable NMR or not.
  • nmr_limit_depth()
    • The engine doesn't execute NMR, if the depth less than this set value.
  • nmr_search_reduction()
    • NMR searches in "(The depth) - (This set value)" as depth.
  • nmr_reduction()
    • A number of reduction of depth if NMR has succeeded.

ProbCut

  • enable_probcut()
    • Set whether to enable ProbCut or not.
  • probcut_limit_depth()
    • The engine doesn't execute ProbCut, if the depth less than this set value.
  • probcut_margin()
    • A increment value of the beta value when ProbCut searching.
  • probcut_search_reduction()
    • ProbCut searches in "(The depth) - (This set value)" as depth.

History Pruning

  • enable_history_pruning()
    • Set whether to enable History Pruning or not.
  • history_pruning_limit_depth()
    • The engine doesn't execute History Pruning, if the depth less than this set value.
  • history_pruning_move_threshold()
    • The engine executes History Pruning after "(Number of candidate moves) * (This set value)" of candidate moves have been searched.
  • history_pruning_after_moves()
    • The engine doesn't execute History Pruning before this set value of candidate moves have been searched, even if history_pruning_move_threshold() is too small.
  • history_pruning_threshold()
    • Reduce the depth of a candidate move which has history value less than "(The max history value) * (This set value)"
  • history_pruning_reduction()
    • A number of reduction of depth.

Late Move Reduction

  • enable_lmr()
    • Set whether to enable LMR or not.
  • lmr_limit_depth()
    • The engine doesn't execute LMR, if the depth less than this set value.
  • lmr_threshold()
    • The engine executes LMR after "(Number of candidate moves) * (This set value)" of candidate moves have been searched.
  • lmr_after_moves()
    • The engine doesn't execute LMR before this set value of candidate moves have been searched, even if lmr_threshold() is too small.
  • lmr_search_reduction()
    • LMR searches in "(The depth) - (This set value)" as depth.

Futility Pruning

  • enable_futility_pruning()
    • Set whether to enable Futility Pruning or not.
  • futility_pruning_depth()
    • Execute Futility Pruning when the depth is less than this set value.
  • futility_pruning_margin()
    • Execute Futility Pruning by "(This set valu) * (The dept)" as margin.
