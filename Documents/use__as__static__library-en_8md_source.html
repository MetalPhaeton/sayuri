1 How to use Sayuri as static library
2 ===================================
3 
4 Summary
5 -------
6 
7 If you use Sayuri's source file as static library, you can embed it into your
8 program.
9 
10 Sayuri is &quot;MIT License&quot; software.
11 To copy and paste the content of Sayuri's &quot;LICENSE&quot; file to your
12 README or LICENSE file, you can freely use Sayuri.
13 
14 Sayuri's special feature is that you can customize the search algorithm or
15 the evaluation function **without changing Sayuri's source code** .
16 
17 
18 
19 Background
20 ----------
21 
22 * Sayuri is written in C++11. So you have to use a compiler supporting C++11
23  language.
24 
25 * Sayuri uses a function of multi-threading of OS.
26  When you compile Sayuri, you have to link multi-threading library to it.
27 
28 * All source files and header file are placed in &quot;src&quot; directory.
29  It doesn't need the other files.
30 
31  You can use all the functions of Sayuri if you include
32  &quot;[sayuri.h](@ref sayuri.h)&quot;.
33 
34 * When you use Sayuri as static library, you have to exclude
35  &quot;[main.cpp](@ref main.cpp)&quot;.
36  Because it contains &quot;main()&quot;.
37 
38 * The namespace of Sayuri is &quot;[Sayuri](@ref Sayuri)&quot;.
39 
40 * Sayuri as static library is also operated by UCI command.
41 
42 * If you have compiled Sayuri with CMake, you can use
43  &quot;libsayurilib.a&quot; and
44  header files in &quot;include&quot; directory in the same way.
45 
46 
47 
48 Usage
49 -----
50 
51 ### Preparation ###
52 
53 First, include &quot;[sayuri.h](@ref sayuri.h)&quot;.
54 
55 ~~~ {.cpp}
56  // Include Sayuri.
57  #include "sayuri.h"
58 ~~~
59 
60 Initialize library before using it.
61 
62 Note: You " **must** " initialize Sayuri before using it.
63 If you don't do so, it won't work well.
64 
65 ~~~ {.cpp}
66  // Initialize Sayuri.
67  Sayuri::Init();
68 ~~~
69 
70 Create the following 2 objects for parameters.
71 
72 * [SearchParams](@ref Sayuri::SearchParams):
73  Parameters for the search algorithm.
74 * [EvalParams](@ref Sayuri::EvalParams):
75  Parameters for the evaluation function.
76 
77 ~~~ {.cpp}
78  // Create SearchParams.
79  std::unique_ptr<Sayuri::SearchParams>
80  search_params_ptr(new Sayuri::SearchParams());
81 
82  // Create EvalParams.
83  std::unique_ptr<Sayuri::EvalParams>
84  eval_params_ptr(new Sayuri::EvalParams());
85 ~~~
86 
87 Create &quot;[ChessEngine](@ref Sayuri::ChessEngine)&quot; object
88 and relate the above 2 objects to it.
89 
90 ~~~ {.cpp}
91  // Create ChessEngine and relate the parameter objects to it.
92  std::unique_ptr<Sayuri::ChessEngine>
93  engine_ptr(new Sayuri::ChessEngine(*search_params_ptr, *eval_params_ptr));
94 ~~~
95 
96 Create &quot;[UCIShell](@ref Sayuri::UCIShell)&quot; object
97 and relate the engine to it.
98 
99 ~~~ {.cpp}
100  // Create UCIShell and relate the engine to it.
101  std::unique_ptr<Sayuri::UCIShell>
102  shell_ptr(new Sayuri::UCIShell(*engine_ptr));
103 ~~~
104 
105 Create a call-back function and register it to
106 &quot;[UCIShell](@ref Sayuri::UCIShell)&quot;.
107 
108 * [UCIShell::AddOutputListener](@ref Sayuri::AddOutputListener):
109  A function to register a call-back function.
110 
111 * &quot;`void MyFunction(const std::string& output)`&quot;:
112  Type of the call-back function.
113 
114 
115 ~~~ {.cpp}
116  // Register the call-back function.
117  shell_ptr->AddOutputListener(MyFunction);
118 ~~~
119 
120 ### To operate the library ###
121 
122 To operate Sayuri, you send UCI commands to
123 &quot;[UCIShell](@ref Sayuri::UCIShell)&quot;.
124 
125 * [UCIShell::InputCommand](@ref Sayuri::UCIShell::InputCommand):
126  A function to send commands to
127  &quot;[UCIShell](@ref Sayuri::UCIShell)&quot;.
128 
129 ~~~ {.cpp}
130  // Create a command.
131  std::string cmd = "uci";
132 
133  // Run the command.
134  shell_ptr->InputCommand(cmd);
135 
136  // Sayuri calls the registered functions and send it the following outputs.
137  //
138  // id name Sayuri ****.**.**
139  // id author Hironori Ishibashi
140  // option name Hash type spin default 32 min 8 max 8192
141  // option name Clear Hash type button
142  // option name Ponder type check default true
143  // option name Threads type spin default 1 min 1 max 64
144  // option name UCI_AnalyseMode type check default false
145  // uciok
146 ~~~
147 
148 ### Example ###
149 
150 ~~~ {.cpp}
151  #include <iostream>
152  #include <string>
153  #include <memory>
154 
155  #include "sayuri.h" // The header file for the library.
156 
157  // The call-back function to recive outputs from the engine.
158  // In this example, the function send the outputs to stdout.
159  void Print(const std::string& message) {
160  std::cout << message << std::endl;
161  }
162 
163  int main(int argc, char* argv[]) {
164  // =========== //
165  // Preparation //
166  // =========== //
167  // Initialize the library. you must do it.
168  Sayuri::Init();
169 
170  // Create SearchParams.
171  std::unique_ptr<Sayuri::SearchParams>
172  search_params_ptr(new Sayuri::SearchParams());
173 
174  // Create EvalParams.
175  std::unique_ptr<Sayuri::EvalParams>
176  eval_params_ptr(new Sayuri::EvalParams());
177 
178  // Create ChessEngine and relate the 2 parameters object to it.
179  std::unique_ptr<Sayuri::ChessEngine>
180  engine_ptr(new Sayuri::ChessEngine(search_params_ptr,
181  eval_params_ptr));
182 
183  // Create UCIShell and relate the engine to it.
184  std::unique_ptr<Sayuri::UCIShell>
185  shell_ptr(new Sayuri::UCIShell(*engine_ptr));
186 
187  // Register the call-back function to the shell.
188  shell_ptr->AddOutputListener(Print);
189 
190  // ================== //
191  // Operate the engine //
192  // ================== //
193  // Command it to the library with InputCommand().
194 
195  shell_ptr->InputCommand("uci");
196  // The engine sends ID, the options, and "uciok" to Print().
197 
198  shell_ptr->InputCommand("isready");
199  // The engine sends "readyok" message to Print().
200 
201  shell_ptr->InputCommand("ucinewgame");
202  shell_ptr->InputCommand("isready");
203  // Prepare to start the new game.
204  // The engine sends "readyok" message to Print().
205 
206  shell_ptr->InputCommand("position startpos");
207  shell_ptr->InputCommand("isready");
208  // Set the position to thinking the next move.
209  // In this example, the position is "startpos",
210  // so it's starting position.
211  // The engine sends "readyok" message to Print().
212 
213  shell_ptr->InputCommand("go depth 10");
214  // Search until the depth of 10 plies.
215  // Return the control soon because the engine is thinking
216  // on the other thread.
217  // The process log is sent to Print().
218  // After finishing searching,
219  // the engine sends the best move to Print()
220  // with "bestmove" command.
221 
222  // Recive commands from stdin.
223  while (true) {
224  std::string input;
225  std::getline(std::cin, input);
226  if (input == "quit") break;
227  else shell_ptr->InputCommand(input);
228  }
229 
230  return 0;
231  }
232 ~~~
