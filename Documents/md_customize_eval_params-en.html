Customize of evaluation function

If you use accessors or mutators of EvalParams, you can change Sayuri's evaluation function.

By default, It's set up by my experience of chess.

Each evaluation score is calculated by "(The weight) * (The value)". And the score of the current node is the total of these scores.

The following constant values are indexes of arrays.

  • NUM_PIECE_TYPES: A number of piece types.
    • EMPTY: It's not any piece.
    • PAWN: Pawn.
    • KNIGHT: Knight.
    • BISHOP: Bishop.
    • ROOK: Rook.
    • QUEEN: Queen.
    • KING: King.
  • NUM_SQUARES: A number of squares on the board.
    • A1: A1 square.
    • B1: B1 square.
    • C1: C1 square.
    • (Omitted from D1 to E8)
    • F8: F8 square.
    • G8: G8 square.
    • H8: H8 square.

Accessors and mutators to value tables

  • opening_position_value_table()
    • A array of values that a specific piece is on a specific square when opening.
  • ending_position_value_table()
    • A array of values that a specific piece is on a specific square when ending.
  • attack_value_table()
    • A array of values that a specific attacker(the first index) is attacking a specific target(the second index).
  • defense_value_table()
    • A array of values that a specific defender(the first index) is guarding a specific friend piece(the second index).
  • pin_value_table()
    • A array of values that a specific attacker(the first index) is pinning a specific opponent's target(the second index) to a specific opponent's piece(the third index).
  • pawn_shield_value_table()
    • A array of values that a specific Pawn which have to guard his King is on a specefic square.
    • When King is in the area of the first or the second rank of A file or B file or C file, a specific Pawn is a Pawn on A file or B file or C file.
    • When King is in the area of the first or the second rank of F file or G file or H file, a specific Pawn is a Pawn on F file or G file or H file.

Weight Object

Weight object is an object to generate evaluation score by multiplying the weight and the value.

This object has 2 parameters, which are an opening(32 pieces exists) weight and an ending(2 Kings only) weight.

Actual weight is calculated by linear function of number of pieces on the board with above parameters.

  • opening_weight()
    • A weight when opening(32 pieces exists).
  • ending_weight()
    • A weight when ending(2 Kings only).

Accessors and mutators to weights

  • weight_opening_position()
    • Weights for opening_position_value_table().
    • You can set weight to each piece type.
    • I recommend "1.0" for opening, "0.0" for ending.
  • weight_ending_position()
    • Weights for ending_position_value_table().
    • You can set weight to each piece type.
    • I recommend "0.0" for opening, "1.0" for ending.
  • weight_mobility()
    • A weight for mobility.
    • The value is number of squares where a piece can move to.
    • You can set weight to each piece type.
  • weight_center_control()
    • A weight for controlling an area has C3 and F3 and C6 and F6 corners.
    • The value is number of squares attacked by a piece in the area.
    • You can set weight to each piece type.
  • weight_sweet_center_control()
    • A weight for controlling D4 or E4 or D5 or E5 square.
    • The value is number of those squares attacked by a piece.
    • You can set weight to each piece type.
  • weight_development()
    • A weight for development of minor pieces.
    • The value is number of Knights and Bishops that are not placed on starting position.
    • You can set weight to each piece type.
  • weight_attack()
    • Weights for attack_value_table().
    • You can set weight to each piece type.
  • weight_defense()
    • Weights for defense_value_table().
    • You can set weight to each piece type.
  • weight_pin()
    • Weights for pin_value_table().
    • You can set weight to each piece type.
  • weight_attack_around_king()
    • A weight for attacking squares around enemy King.
    • The value is number of squares around enemy King attacked by a piece.
    • You can set weight to each piece type.
  • weight_pass_pawn()
    • A weight for Pass Pawns.
    • The value is number of Pass Pawns(no enemy pawns on 3 files in front of it).
  • weight_protected_pass_pawn()
    • A weight for Protected Pass Pawns.
    • The value is number of Pass Pawns that are protected his fiend Pawns.
    • This is bonus point to add to weight_pass_pawn().
  • weight_double_pawn()
    • A weight for Double Pawn.
    • The value is number of Double Pawns(exists more than 2 Pawns on same file).
    • The value is never 1, because Double Pawn needs at least 2 Pawns.
    • This is supposed as bad strategy. So I recommend to set negative number.
  • weight_iso_pawn()
    • A weight for Isolated Pawn.
    • The value is number of Isolated Pawns(no friend Pawns on the adjoined files).
    • This is supposed as bad strategy. So I recommend to set negative number.
  • weight_pawn_shield()
    • A weight for pawn_shield_value_table().
    • I recommend "0.0" or "1.0".
  • weight_bishop_pair()
    • A weight for existing more than 2 Bishops.
    • The value is 1 if there are more than 2 Bishops.
    • The value is not increased, even if there are more than 3 Bishops
  • weight_bad_bishop()
    • A weight for Bad Bishop.
    • The value is number of Pawns on the same colored square of a Bishop.
    • This is supposed as bad strategy. So I recommend to set negative number.
  • weight_rook_pair()
    • A weight for existing more than 2 Rooks.
    • The value is 1 if there are more than 2 Rooks.
    • The value is not increased, even if there are more than 3 Rooks
  • weight_rook_semiopen_fyle()
    • A weight for Rook on semi-open file.
    • The value is number of Rooks on semi-open file.
    • For my reason, I spell "file" to "fyle". It's not mistakes.
  • weight_rook_open_fyle()
    • A weight for Rook on open file.
    • The value is number of Rooks on open file.
    • For my reason, I spell "file" to "fyle". It's not mistakes.
  • weight_early_queen_launched()
    • A weight for Queen developed too early time.
    • The value is number of Knights and Bishops on starting position when Queen is not placed on its starting position.
    • This is supposed as bad strategy. So I recommend to set negative number.
  • weight_weak_square()
    • A weight for weak squares around King.
    • The value is number of squares that no pawn is placed on and the same colored of opponent's Bishop and in the specific area.
    • The specific area is the second rank or the third rank of A file or B file or C file when King on the first rank or the second rank of A file or B file or C file.
    • The specific area is the second rank or the third rank of F file or G file or H file when King on the first rank or the second rank of F file or G file or H file.
    • This is supposed as bad strategy. So I recommend to set negative number.
  • weight_castling()
    • A weight for Castling.
    • The value is 1 after King has castled.
    • This engine judges whether King has castled or not by a castling move. So if King has already castled before game has started with FEN, the engine judges King has abandoned castling rights.
  • weight_abandoned_castling()
    • A weight for abandoned castling rights.
    • The value is 1 after castling rights has lost without castling move.
    • This engine judges whether King has castled or not by a castling move. So if King has already castled before game has started with FEN, the engine judges King has abandoned castling rights.
    • This is supposed as bad strategy. So I recommend to set negative number.
