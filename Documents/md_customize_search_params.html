探索アルゴリズムのカスタマイズ

SearchParamsオブジェクトの アクセサ・ミューテータを使うことで、探索アルゴリズムを変更することができます。

デフォルトでは、作者の「根拠のない勘」で設定されています。

配列のインデックスで使用する値は以下のとおりです。

  • NUM_PIECE_TYPES: 駒の種類の数。
    • EMPTY: どの駒でもない。
    • PAWN: ポーン。
    • KNIGHT: ナイト。
    • BISHOP: ビショップ。
    • ROOK: ルーク。
    • QUEEN: クイーン。
    • KING: キング。
  • NUM_SQUARES: ボード上のマスの数。
    • A1: a1 のマス。
    • B1: b1 のマス。
    • C1: c1 のマス。
    • (D1 から E8 まで中略)
    • F8: f8 のマス。
    • G8: g8 のマス。
    • H8: h8 のマス。

アクセサ・ミューテータ

マテリアル

  • material()
    • チェスエンジンで最も重要な数値、マテリアルを設定します。
    • 探索アルゴリズムだけでなく、評価関数でも使用します。

YBWC

  • ybwc_limit_depth()
    • 探索の残り深さが設定値未満になら、YBWCをしません。
  • ybwc_after_moves()
    • 現在の探索ノードの候補手を設定値の数だけ探索した後にYBWCを実行します。

Aspiration Windows

  • enable_aspiration_windows()
    • Aspration Windowsの機能を有効にするかどうかを設定します。
  • aspiration_windows_limit_depth()
    • 探索の残り深さが設定値未満になら、Aspiration Windowsをしません。
  • aspiration_windows_delta()
    • 探索ウィンドウのサイズを広げていく基本の値。
    • 一度失敗するごとに、「設定値 * 2n」 ずつ広げていく。

SEE

  • enable_see()
    • Move OrderingでSEEの機能を有効にするかどうかを設定します。

History Heuristic

  • enable_history()
    • History Heuristicの機能を有効にするかどうかを設定します。

Killer Move Heuristic

  • enable_killer()
    • Killer Move Heuristicの機能を有効にするかどうかを設定します。
  • enable_killer_2()
    • 2プライ前のキラームーブを有効にするかどうかを設定します。

Transposition Table

  • enable_ttable()
    • トランスポジションテーブルを有効にするかどうかを設定します。

Internal Iterative Deepening

  • enable_iid()
    • Internal Iterative Deepeningを有効にするかどうかを設定します。
  • iid_limit_depth()
    • 探索の残り深さが設定値未満になら、IIDをしません。
  • iid_search_depth()
    • IIDで探索する深さ。

Null Move Reduction

  • enable_nmr()
    • Null Move Reductionを有効にするかどうかを設定します。
  • nmr_limit_depth()
    • 探索の残り深さが設定値未満になら、NMRをしません。
  • nmr_search_reduction()
    • NMRで「現在の残り深さ - 設定値」の深さで探索します。
  • nmr_reduction()
    • NMRが成功した時、探索の残り深さを減らす量。

ProbCut

  • enable_probcut()
    • ProbCutを有効にするかどうかの設定をします。
  • probcut_limit_depth()
    • 探索の残り深さが設定値未満になら、ProbCutをしません。
  • probcut_margin()
    • ProbCutで探索する際の、ベータ値の増分。
  • probcut_search_reduction()
    • ProbCutで「現在の残り深さ - 設定値」の深さで探索します。

History Pruning

  • enable_history_pruning()
    • History Pruningを有効にするかどうかの設定をします。
  • history_pruning_limit_depth()
    • 探索の残り深さが設定値未満になら、History Pruningをしません。
  • history_pruning_move_threshold()
    • 「現在の探索ノードの候補手の数 * 設定値」の数の探索を終えた後、 History Pruningを実行します。
  • history_pruning_after_moves()
    • 「history_pruning_move_threshold()」の値がどんなに小さくても、 現在の探索ノードの候補手を設定値の数だけはHistory Pruningをしません。
  • history_pruning_threshold()
    • 「最大ヒストリー値 * 設定値」以下のヒストリー値の候補手の 残り深さを減らします。
  • history_pruning_reduction()
    • History Pruningで残り深さを減らす量。

Late Move Reduction

  • enable_lmr()
    • LMRを有効にするかどうかの設定をします。
  • lmr_limit_depth()
    • 探索の残り深さが設定値未満になら、LMRをしません。
  • lmr_threshold()
    • 「現在の探索ノードの候補手の数 * 設定値」の数の探索を終えた後、 LMRを実行します。
  • lmr_after_moves()
    • 「lmr_threshold()」の値が どんなに小さくても、 現在の探索ノードの候補手を設定値の数だけはLMRをしません。
  • lmr_search_reduction()
    • LMRで「現在の残り深さ - 設定値」の深さで探索します。

Futility Pruning

  • enable_futility_pruning()
    • Futility Pruningを有効にするかどうかの設定をします。
  • futility_pruning_depth()
    • 残り深さが設定値以下の時にFutility Pruningを実行します。
  • futility_pruning_margin()
    • 「設定値 * 残り深さ」をマージンにして実行します。
