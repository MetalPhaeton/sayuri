42 CMakeを使ったビルド
43 -------------------
44 
45 CMakeを使って簡単にビルドすることができます。
46 
47 使用するコンパイラはデフォルトで「clang++」となります。
48 (コンパイラを変更したいときは、「CMakeLists.txt」の26行目、
49 27行目のコンパイラ設定を変更してください。)
50 
51 ###### 通常のビルド手順 ######
52 
53 1. 「CMakeLists.txt」のあるディレクトリに移動します。
54 2. 以下のコマンドを実行します。
55  1. $ mkdir build
56  2. $ cd build
57  3. $ cmake ..
58  4. $ make
59 3. 「build」ディレクト内に以下のファイル、ディレクトリが完成します。
60  * sayuri : UCI用チェスエンジン。
61  * libsayurilib.a : 静的ライブラリ。
62  * include : 静的ライブラリ用ヘッダファイルの入ったディレクトリ。
63 
64 ###### デバッグモードでのビルド手順 ######
65 
66 デバッグモードでビルドすと、「最適化なし」、「デバッグ情報の埋め込み」で
67 ビルドすることができます。
68 
69 以下の手順2-3で「-DCMAKE\_BUILD\_TYPE=Debug」を挿入するだけです。
70 
71 1. 「CMakeLists.txt」のあるディレクトリに移動します。
72 2. 以下のコマンドを実行します。
73  1. $ mkdir build
74  2. $ cd build
75  3. $ cmake -DCMAKE\_BUILD\_TYPE=Debug ..
76  4. $ make
77 3. 「build」ディレクト内に以下のファイル、ディレクトリが完成します。
78  * sayuri : UCI用チェスエンジン。
79  * libsayurilib.a : 静的ライブラリ。
80  * include : 静的ライブラリ用ヘッダファイルの入ったディレクトリ。
81 
82 ###### CMakeで作成した各ファイルについて ######
83 
84 「sayuri」はUCI用チェスエンジンです。
85 
86 このエンジンを使用する場合は、「libsayurilib.a」と
87 「include」ディレクトリは必要ありません。
88 
89 「libsayurilib.a」と「include」はSayuriを静的ライブラリとして
90 使用する場合に使います。その場合は「sayuri」は必要ありません。
91 
92 詳しくは「[静的ライブラリとしての利用法](@ref use_as_static_library.md)」
93 を読んで下さい。

112 UCIオプション
113 -------------
114 
115 以下のコマンドでエンジンの設定を変更できます。
116 
117 * ハッシュテーブルのサイズを変更。 (デフォルトは 32。最大は 8192。最小は 8。)
118  * setoption name Hash value <サイズ、メガバイト>
119 
120 * ハッシュテーブルを初期化。
121  * setoption name Clear Hash
122 
123 * Ponderの有効化、無効化。 (デフォルトは true。)
124  * setoption name Ponder value <true、又はfalse>
125 
126 * 探索用スレッドの数を変更。 (デフォルトは 1。最大は 64。最小は 1。)
127  * setoption name Threads value <スレッドの数>
128 
129 * アナライズモードの有効化、無効化。 (デフォルトは false。)
130  * setoption name UCI\_AnalyseMode value <true、又はfalse>
